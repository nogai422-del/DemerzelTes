# Быстрый запуск на Bothost

- Рабочая директория кода внутри образа: `/usr/src/app`.
- Постоянная база: `/app/data/database.db`.
- Volume должен монтироваться именно в `/app/data`, не в `/usr/src/app` и не во весь `/app`.
- Команда запуска уже указана в Dockerfile: `python demerzel.py`.
- Приложение слушает `0.0.0.0:$PORT`.

Перед первым запуском новой версии обязательно остановите старую копию бота. Один Telegram-токен не может одновременно использоваться двумя `getUpdates`-процессами.

После запуска откройте `/health`. Нормальный ответ новой версии:

```json
{"status":"ok","missing_tables":[],"form_filling_templates":12}
```

При проблеме база вернёт список отсутствующих таблиц или неполное количество промежуточных шаблонов, а traceback HTTP 500 будет в журнале Bothost.

## Картинки оповещений

Новые изображения этапов анкеты и донат-уведомлений сохраняются в:

- `/app/data/uploads/permission_images`
- `/app/data/uploads/donation_images`

Они находятся в том же постоянном volume, что и база данных, и не исчезают после пересборки. При резервном копировании сохраняйте всю папку `/app/data`, а не только `database.db`.

Обновлённый `/health` также возвращает:

```json
{
  "status": "ok",
  "missing_tables": [],
  "form_filling_templates": 12,
  "chat_user_duplicates": 0,
  "notification_images_missing": 0
}
```

`notification_images_missing` может быть больше нуля только если в базе осталась ссылка на отсутствующий файл. При старте такие ссылки автоматически очищаются; текст уведомления продолжает работать в любом случае.
