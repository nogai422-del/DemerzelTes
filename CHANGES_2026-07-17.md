# Расширение функционала — 17 июля 2026

## Три этапа анкеты

Теперь бот различает состояния пользователя:

1. `new` — анкету ещё не отправляли командой `/bv`;
2. `filling` — анкета отправлена, но ещё не сохранена через `/save`;
3. `saved` — анкета успешно сохранена.

Для этапа `filling` добавлены отдельные настраиваемые предупреждения для:

- мата;
- смайликов;
- ссылок;
- фотографий;
- видео;
- GIF;
- аудио;
- файлов;
- голосовых сообщений;
- кружков;
- стикеров;
- реакций.

Команда `/bv` переводит нового пользователя в этап заполнения. Команда `/save` завершает этап, удаляет запись активной `/bv` и включает обычные уведомления ограничений.

## Админ-панель

- Боковое меню пересобрано по смысловым группам.
- Главная страница разделена на блоки: пользователи, анкета, модерация, донаты, контент и интерфейс.
- Добавлена сводка по этапам анкеты.
- Страница уведомлений разделена на три понятных этапа.

## Участники

- Фильтрация по чату, этапу анкеты, статусу Telegram и активному донату.
- Поиск по имени, username и Telegram ID.
- Сортировка и постраничный вывод.
- Отображение активных донатов и срока действия.
- Быстрая выдача доната из строки участника.
- Копирование Telegram ID одним нажатием.
- CSV-импорт помещён в отдельный раскрывающийся блок.
- После выдачи доната через панель бот обязательно отправляет уведомление в выбранный чат.

## База данных

В `chat_users` добавлены поля:

- `form_stage`;
- `form_started_at`;
- `form_saved_at`.

Миграция сохраняет старые данные:

- запись с заполненным `filled_form_text` переводится в `saved`;
- пользователь с активной записью `/bv` переводится в `filling`;
- остальные записи остаются в `new`.

## Диагностика

`/health` дополнительно проверяет таблицы участников, донатов, уведомлений и наличие 12 промежуточных шаблонов.

## Интеграция свежей рабочей базы

- Встроенная база заменена предоставленной свежей версией.
- Сохранены 7 523 записи `chat_users`, 28 выдач донатов и 58 записей использования донатов.
- Поверх свежих данных добавлены технические таблицы и столбцы текущей версии проекта.
- 7 314 уникальных пользователей автоматически добавлены в каталог `admin_members` по Telegram ID; CSV может дополнить имена, username и статусы без дублей.
- При первом запуске на пустом `/app/data` теперь копируется вся встроенная база, а не только таблицы настроек.
- Существующая непустая база на volume автоматически не перезаписывается.
- Стандартный шаблон предупреждения о кружках за 3 дня переведён на переменную `{days_left}`, чтобы учитывать настройку срока предварительного уведомления.
