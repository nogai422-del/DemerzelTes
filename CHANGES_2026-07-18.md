# Изменения 18.07.2026 — уведомления после /save

## Исправлено

- Устранены тихие удаления сообщений после `/save`: неизвестные Telegram-типы теперь используют общий шаблон `other_media`.
- Добавлен отдельный редактируемый шаблон `sticker` для стикеров после `/save`.
- В админ-панели шаблоны «Стикеры» и «Другой тип вложения» отображаются в этапе 3.
- Кнопочные URL вида `@username` автоматически преобразуются в `https://t.me/username`.
- Исправлены сохранённые кнопки «Написать Найту» в уведомлениях ГС и смайликов.
- Уведомления ограничений и донатов получили каскадный fallback:
  1. изображение + HTML + кнопки;
  2. HTML + кнопки;
  3. HTML без кнопок;
  4. обычный текст без форматирования.
- Ошибка или отсутствие изображения больше не отменяет текст уведомления.
- GIF в уведомлениях ограничений отправляются через `sendAnimation`, а не через `sendPhoto`.
- Изображение для уведомления об эмодзи теперь также отправляется, если оно настроено.
- Даже при случайном удалении строки шаблона из SQLite бот использует встроенный текст и не молчит.

## Текущие уровни медиа

### Медиа 1

- фотографии;
- обычные видео;
- GIF и Telegram-анимации.

### Медиа 2

Всё из Медиа 1, а также:

- документы и файлы;
- музыка и аудиофайлы;
- реакции;
- ссылки;
- ругательства и слова из списка модерации.

Не входят в Медиа 1/2 и выдаются отдельно:

- голосовые сообщения;
- смайлики и кастомные эмодзи;
- видеокружки;
- тег;
- право просмотра анкет.

Стикеры сейчас запрещены на любом уровне и получают отдельное предупреждение.

## Дополнительное исправление — картинки на всех этапах

- Добавлен единый модуль `bot/notification_delivery.py` для уведомлений этапов до `/bv`, между `/bv` и `/save`, после `/save`, а также донат-уведомлений.
- Картинка стала необязательным дополнением: ошибка файла больше не может отменить текст.
- Старое предупреждение удаляется только после успешной отправки нового.
- JPG, PNG и WEBP автоматически преобразуются в безопасный JPEG размером до 1920 px.
- Прозрачность заменяется белым фоном, EXIF-ориентация исправляется, экстремальные пропорции дополняются полями.
- Анимированный GIF сохраняется как анимация и получает запасной статичный JPEG-кадр.
- Повреждённые, пустые, слишком большие и подозрительно большие по разрешению файлы отклоняются ещё при загрузке.
- Длинный текст, который не помещается в подпись Telegram, сразу отправляется обычным сообщением.
- При ошибке медиа выполняются попытки: безопасное фото/GIF, вариант без кнопок, HTML-текст, HTML без кнопок, обычный текст.
- Все новые картинки предупреждений сохраняются в постоянном `DATA_DIR/uploads` (`/app/data/uploads` на Bothost), поэтому не исчезают после пересборки контейнера.
- Старые мёртвые ссылки на исчезнувшие файлы автоматически очищаются без изменения текстов и кнопок.
- Картинки теперь разрешены и для уведомлений об эмодзи на всех трёх этапах.
- В админ-панели для каждого этапа добавлено явное удаление текущей картинки.
- Допустимые форматы загрузки: JPG, PNG, WEBP и GIF до 20 МБ.
- Исправлены 87 групп дублирующихся записей `chat_users`; добавлен уникальный индекс `(chat_id, user_id)`.
- Этап `saved` теперь определяется и по полю `form_stage`, даже если старый импорт не содержит текста анкеты.
- `/health` дополнительно показывает `chat_user_duplicates` и `notification_images_missing`.
