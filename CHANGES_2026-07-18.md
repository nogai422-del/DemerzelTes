# Изменения 18.07.2026 — уведомления после /save

## Исправлено

- Устранены тихие удаления сообщений после `/save`: неизвестные Telegram-типы теперь используют общий шаблон `other_media`.
- Добавлен отдельный редактируемый шаблон `sticker` для стикеров после `/save`.
- В админ-панели шаблоны «Стикеры» и «Другой тип вложения» отображаются в этапе 3.
- Кнопочные URL вида `@username` автоматически преобразуются в `https://t.me/username`.
- Исправлены сохранённые кнопки «Написать Найту» в уведомлениях ГС и смайликов.
- Уведомления ограничений и донатов получили каскадный fallback:
  1. изображение + HTML + кнопки;
  2. HTML + кнопки;
  3. HTML без кнопок;
  4. обычный текст без форматирования.
- Ошибка или отсутствие изображения больше не отменяет текст уведомления.
- GIF в уведомлениях ограничений отправляются через `sendAnimation`, а не через `sendPhoto`.
- Изображение для уведомления об эмодзи теперь также отправляется, если оно настроено.
- Даже при случайном удалении строки шаблона из SQLite бот использует встроенный текст и не молчит.

## Текущие уровни медиа

### Медиа 1

- фотографии;
- обычные видео;
- GIF и Telegram-анимации.

### Медиа 2

Всё из Медиа 1, а также:

- документы и файлы;
- музыка и аудиофайлы;
- реакции;
- ссылки;
- ругательства и слова из списка модерации.

Не входят в Медиа 1/2 и выдаются отдельно:

- голосовые сообщения;
- смайлики и кастомные эмодзи;
- видеокружки;
- тег;
- право просмотра анкет.

Стикеры сейчас запрещены на любом уровне и получают отдельное предупреждение.
